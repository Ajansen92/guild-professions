Connect a shared database

A free Supabase project gives the whole guild one synced roster. Takes about 3 minutes.

  1. Create a free project at supabase.com (supabase.com).
  2. In the SQL below, change changeme to your guild passphrase (keep the quotes). Then open the SQL Editor, paste, and run:
    create table if not exists characters ( id text primary key, name text not null, cls text, profs jsonb default '[]', notes text, updated bigint ); alter table characters enable row level security; -- Anyone may READ the roster drop policy if exists "guild access" on characters; drop policy if exists "read roster" on characters; create policy "read roster" on characters for select using (true); grant select on characters to anon; -- No write policy: direct edits are blocked. Writes go through the -- functions below, which require the passphrase. -- >>> set your guild passphrase here (used by all three functions) <<< create or replace function guild_check(pass text) returns boolean language sql security definer set search_path = public as $$ select pass is not distinct from 'changeme'; $$; create or replace function guild_upsert(pass text, c jsonb) returns void language plpgsql security definer set search_path = public as $$ begin if not guild_check(pass) then raise exception 'Wrong guild passphrase'; end if; insert into characters (id,name,cls,profs,notes,updated) values (c->>'id', c->>'name', c->>'cls', coalesce(c->'profs','[]'::jsonb), c->>'notes', (c->>'updated')::bigint) on conflict (id) do update set name=excluded.name, cls=excluded.cls, profs=excluded.profs, notes=excluded.notes, updated=excluded.updated; end; $$; create or replace function guild_delete(pass text, cid text) returns void language plpgsql security definer set search_path = public as $$ begin if not guild_check(pass) then raise exception 'Wrong guild passphrase'; end if; delete from characters where id = cid; end; $$; grant execute on function guild_check(text) to anon; grant execute on function guild_upsert(text,jsonb) to anon; grant execute on function guild_delete(text,text) to anon;
  3. Go to Project Settings → API Keys and copy the Project URL and your publishable key (sb_publishable_…) — or the legacy anon / public key if that's what your project shows. Both work here. Never use the secret or service_role key. Paste the URL and key into the fields below.
Everyone can view and search the roster. Adding or removing a character requires the guild passphrase — and that's enforced inside the database, so it can't be bypassed from the browser. Share the passphrase with officers (or the whole guild) as you see fit.
